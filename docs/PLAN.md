# 주간 식단 추천 앱 — 계획서 (PLAN)

> 최종 갱신: 2026-06-29
> 이 문서는 "무엇을/왜" 만드는지를 다룬다. "어떻게(규칙·스키마)"는 [RULES.md](./RULES.md) 참고.

## 1. 한 줄 정의
7일치(점심·저녁) 식단을 추천하고, 메뉴를 누르면 레시피와 필요한 재료를 보여주며,
한 주 전체 재료를 모아 카테고리별 장바구니(체크리스트 + 검색 링크)로 정리해 주는 개인용 웹앱.

## 2. 핵심 사용자 시나리오
1. 로그인한다.
2. "이번 주 식단" 화면에서 7일 × 2끼(점심·저녁) = 총 14개 칸을 본다.
3. 각 칸마다 **5개의 메뉴 선지**가 제시된다. (집밥 4 + 외식 1 권장 구성)
   - 선지는 한식/중식/일식/양식 + 외식이 골고루 섞이되, **전날 먹은 요리 종류는 제외**된다.
4. 마음에 드는 게 없으면 **갱신 버튼**으로 새로운 5개 선지를 받는다.
5. 메뉴를 선택하면 그 칸이 확정된다.
6. 메뉴를 누르면 **레시피 상세**(조리 단계 + 재료 목록)를 본다.
7. 14칸을 다 채우면 **장바구니** 화면에서 한 주 재료가 합산·중복 제거되어
   카테고리별 체크리스트로 보이고, 항목마다 외부 검색 링크(쿠팡/마켓컬리/네이버)가 붙는다.
8. 앱은 선택/갱신/좋아요 기록을 학습해 다음 추천을 점점 사용자 취향에 맞춘다.

## 3. 기술 스택 (확정)
| 레이어 | 선택 | 비고 |
|---|---|---|
| 프론트엔드 | React + Vite + TypeScript, PWA | Windows에서 즉시 개발/테스트, 폰 홈화면 설치 |
| 스타일 | **Tailwind CSS** | 확정 |
| 상태관리 | React Query(서버상태) + 가벼운 클라이언트 상태 | |
| 백엔드 | Supabase | Auth + Postgres + Edge Functions |
| 인증 | Supabase Auth (이메일 + 구글) | |
| AI | Claude API · 기본 **`claude-opus-4-8`** | **Edge Function 안에서만 호출** (키 보호) |
| 메뉴 생성 | 온디맨드 + DB 캐싱 | '갱신' 또는 캐시 미스 시에만 호출 |
| 배포 | 프론트: Vercel/Netlify · 백엔드: Supabase 클라우드 | |

## 4. 시스템 구성 (데이터 흐름)
```
[브라우저(PWA)]
   │  메뉴 5개 요청(슬롯 제약: 요일/끼니/제외요리/취향가중치)
   ▼
[Supabase Edge Function: generate-menus]
   │  1) menu_cache 조회 (제약 키로) → 있으면 반환
   │  2) 없거나 '갱신'이면 → Claude API 호출 → JSON 검증 → menu_cache 저장
   ▼
[Postgres]  preference_profiles / meal_plans / meal_slots / menu_options
            / menu_cache / selection_events / shopping_list
```
- Claude API 키는 Edge Function 환경변수에만 둔다. 프론트에는 절대 노출하지 않는다.

## 5. 데이터 모델 (개요 — 상세 스키마는 RULES.md / 0001_init.sql)
**날짜 기반 모델 (RULES R8)** — 고정 주간 그리드가 아니라 날짜별 엔트리 누적.
- `preference_profiles` — 사용자별 취향(요리별 가중치, 비선호 재료, 알레르기)
- `meal_entries` — **(날짜, 끼니)에 확정 저장된 한 끼.** 핵심 테이블. 선택 시 upsert, 안 한 끼니는 행 없음(자동 제외). 주 단위 누적 히스토리의 원천.
- `selection_events` — 학습용 행동 로그(선택/스킵/좋아요/싫어요)
- `menu_cache` — AI 생성 보충분 캐시(시드로 부족할 때만)
- `shopping_checks` — 장바구니 체크 상태 유지(항목은 `meal_entries`에서 합산)
- **시드 풀** `shared/seed/` — 120+개 기본 레시피(R9), 선지·갱신의 1차 출처(코드 내장, DB 아님)

## 6. 화면 목록 (MVP)
1. **로그인/회원가입** — Supabase Auth(이메일+구글). 비로그인은 여기로(R8-1).
2. **식단(홈)** — **오늘 날짜부터** 날짜별(요일 표시) 점심/저녁 슬롯. 슬롯당 5선지 + 갱신, 선택 시 즉시 저장(엔트리). 오늘+향후 13일 롤링(R8-2).
3. **히스토리** — 주(월요일 시작) 단위로 과거 확정 식단 누적 조회(R8-5).
4. **메뉴 상세** — 레시피 단계 + 재료 목록 + 좋아요/싫어요
5. **장바구니** — 활성 주(또는 날짜범위)의 확정 엔트리 재료 합산 체크리스트 + 검색 링크 + 기본양념 접기, 체크 상태 유지(R8-6)
6. **설정/취향** — 알레르기·비선호 재료, 인분 수, 학습 초기화
